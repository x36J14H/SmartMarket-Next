# Auth API (Аутентификация)

Base URL: `http://<host>:<port>/smartmarket/hs/site-auth`

Регистрация, вход, управление паролем и профилем пользователя.

---

## POST /login

Вход по email и паролю. Возвращает токен сессии.

### Тело запроса

```json
{
  "email": "user@example.com",
  "password": "Secret123!"
}
```

### Ответ `200`

```json
{
  "token": "abc123...",
  "expires_in": 2592000
}
```

`expires_in` — время жизни токена в секундах (30 дней). Сессия скользящая — продлевается при каждом обращении.

### Ответ `400`

```json
{ "error": "Укажите email и password" }
```

### Ответ `401`

```json
{ "error": "Неверный логин или пароль" }
```

## POST /logout

Инвалидация токена (выход из системы).

### Заголовки

| Заголовок      | Значение |
|----------------|----------|
| `X-Auth-Token` | Активный токен |

### Ответ `200`

```json
{ "ok": true }
```

## GET /profile

Профиль текущего пользователя по токену.

### Заголовки

| Заголовок      | Значение |
|----------------|----------|
| `X-Auth-Token` | Активный токен |

### Ответ `200`

```json
{
  "id": "015a375c-9f9c-437d-8c6e-9d49adf7190e",
  "name": "Иван Иванов",
  "email": "user@example.com",
  "phone": "+79001234567",
  "delivery_address": "г. Москва, ул. Ленина, д. 1",
  "registered_at": "2024-01-15T10:00:00Z"
}
```

### Ответ `401`

```json
{ "error": "Токен не передан" }
```

### Ответ `404`

```json
{ "error": "Пользователь не найден" }
```

## POST /register

Регистрация нового пользователя. На email отправляется 6-значный код подтверждения (действует 15 минут). Пользователь создаётся только после подтверждения через `/confirm-email`.

### Тело запроса

```json
{
  "email": "user@example.com",
  "password": "Secret123!",
  "name": "Иван Иванов"
}
```

`name` — необязательное поле.

**Требования к паролю:** минимум 8 символов, хотя бы одна заглавная буква, одна цифра и один спецсимвол (`!@#$%^&*` и др.).

### Ответ `200`

```json
{ "ok": true }
```

### Ответ `400`

```json
{ "error": "Email уже зарегистрирован" }
```

---

## POST /confirm-email

Подтверждение email по коду из письма. Создаёт пользователя и возвращает токен — пользователь сразу залогинен.

### Тело запроса

```json
{
  "email": "user@example.com",
  "code": "123456"
}
```

### Ответ `200`

```json
{
  "token": "abc123...",
  "expires_in": 2592000
}
```

### Ответ `400`

```json
{ "error": "Неверный код" }
```

---

## POST /forgot-password

Отправка кода сброса пароля на email. Всегда возвращает `200` — не раскрывает наличие email в базе.

### Тело запроса

```json
{
  "email": "user@example.com"
}
```

### Ответ `200`

```json
{ "ok": true }
```

---

## POST /reset-password

Установка нового пароля по коду из письма. После успеха все старые сессии инвалидируются, возвращается новый токен.

### Тело запроса

```json
{
  "email": "user@example.com",
  "code": "123456",
  "new_password": "NewSecret123!"
}
```

### Ответ `200`

```json
{
  "token": "abc123...",
  "expires_in": 2592000
}
```

### Ответ `400`

```json
{ "error": "Код истёк. Запросите новый" }
```

---

## POST /change-password

Смена пароля для залогиненного пользователя. Требует текущий пароль.

### Заголовки

| Заголовок      | Значение |
|----------------|----------|
| `X-Auth-Token` | Активный токен |

### Тело запроса

```json
{
  "current_password": "Secret123!",
  "new_password": "NewSecret123!"
}
```

### Ответ `200`

```json
{ "ok": true }
```

### Ответ `400`

```json
{ "error": "Неверный текущий пароль" }
```

---

## Коды ответов

| Код | Описание |
|-----|----------|
| `200` | Успех |
| `400` | Ошибка валидации |
| `401` | Токен не передан или недействителен |
| `404` | Пользователь не найден |
